# Handoff: Pickhouse — 집 보러 다니기 기록·비교 앱 (메인 / 집 추가 / 직장·학교 등록)

## Overview
Pickhouse는 집을 구할 때 여러 매물(보통 5~10곳)을 돌아보며 기록하고, 한눈에 비교하기 위한 모바일 앱이다.
메모장에 적으면 세로로 길게 스크롤해야 해서 비교가 어렵다는 문제를 풀기 위한 앱이다.

이 핸드오프는 아래 **3개 화면군**의 디자인을 담는다.
1. **메인 화면** — 지도(네이버/야놀자 스타일) + 하단 매물 카드 캐러셀 + 하단 탭바. 카드 스와이프 ↔ 지도 핀 동기화. 줌 레벨별 핀 표현. 필터 시트.
2. **집 추가** — 비선형 4탭(기본/가격/구조/체크) 입력 위저드. 조건부 필드. 주소 검색(우편번호 API 스타일).
3. **마이페이지 › 직장·학교 등록** — 통근 기준지 등록(카카오 장소 검색). 등록 시 매물 카드/비교에 통근시간 표시.

> 비교 화면·상세 보기는 아직 디자인되지 않았다(향후 작업). 단, 데이터 모델은 통근시간 축까지 비교에 쓰일 수 있도록 설계되어 있다.

## About the Design Files
이 번들의 파일들은 **HTML로 만든 디자인 레퍼런스**다 — 의도한 모양과 동작을 보여주는 프로토타입이며, 그대로 복사해 프로덕션에 넣을 코드가 아니다.
작업은 이 HTML 디자인을 **타깃 코드베이스의 기존 환경**(React Native, Flutter, SwiftUI, 네이티브 등)에서 그 환경의 관례·라이브러리로 **재구현**하는 것이다. 아직 환경이 없다면, 프로젝트에 가장 적합한 프레임워크를 골라 구현한다.

프로토타입은 React 18 + 인라인 스타일(JSX, Babel standalone)로 작성되었고, `design-canvas.jsx`(여러 시안을 나란히 보는 캔버스)와 `ios-frame.jsx`(아이폰 프레임/상태바)는 **프레젠테이션용 래퍼일 뿐 실제 앱 코드가 아니다** — 무시하고, `Screen`/화면 컴포넌트 내부 구현만 참고하면 된다.

## Fidelity
**High-fidelity (hifi)** — 최종 색상·타이포·간격·인터랙션이 정해진 픽셀 단위 목업이다. 개발자는 코드베이스의 기존 라이브러리·패턴으로 UI를 픽셀 단위로 재현해야 한다. 단, 지도 자체는 스타일라이즈된 가짜 SVG이며 **실제 지도 SDK(네이버맵/카카오맵)로 대체**한다.

## Screens / Views

### 1. 메인 화면 (`pickhouse-main.jsx` → `MainScreen`)
**Purpose**: 앱 진입 첫 화면. 내 위치 근처 내가 기록한 집들을 지도 + 카드로 보여준다. 비교/점수는 여기서 하지 않는다(별도 메뉴).

**Layout (위→아래):**
- **상단 바** (`TopBar`): 좌측 큰 검색 pill(메뉴 아이콘 + "장소, 지명, 집 이름 검색" 플레이스홀더 + 우측 그린 원형 검색 버튼 36px) + 우측 **필터 아이콘 버튼**(48px 원형, sliders 아이콘). 필터 적용 시 버튼이 ink로 채워지고 우상단 coral 카운트 뱃지. **상단엔 칩 줄을 두지 않는다**(필터는 아이콘으로만 진입 — 지도 필터링 오해 방지).
- **지도** (full-bleed): 배경 mapBg. 노란 간선도로 + 흰 보조도로 + 건물 블록 + 공원/하천 + 동 이름 라벨. 실제 구현은 지도 SDK로 대체.
- **핀** (`MapLayer`): 줌 레벨별로 다르게(아래 "줌 레벨" 참조). 기본(medium)은 **물방울 하우스 핀**(coral, 흰 집 아이콘, 흰 2.5px 보더, 같은 위치 다수면 우하단 카운트 뱃지). **선택된 집은 핀 대신 흰 콜아웃 말풍선**(집 아이콘 사각 + 이름 + 가격, 아래쪽 삼각 꼬리, coral 보더).
- **플로팅 버튼**: 지도 우측에 레이어/현위치 버튼(40px 원형, 흰 배경), 중앙에 "목록 보기" pill(ink 배경). 모두 카드 캐러셀 **위에** 위치(겹치지 않게 bottom≈258–262).
- **하단 카드 캐러셀** (`FloatingCards`): 지도 위에 떠있는 가로 스크롤 카드(흰 배경, 다음 카드가 살짝 보이는 peek). `scroll-snap-type: x mandatory`. **스크롤 위치 → 활성 인덱스 → 지도의 선택 핀 동기화(양방향)**. 카드 폭 326, 갭 12.
- **하단 탭바** (`BottomNav`): 5탭 — 지도(navigate) / 목록(list) / **＋집 추가(가운데, 그린 46px 라운드 버튼으로 강조)** / 비교(grid) / 마이(user). 활성 탭은 ink, 나머지 muted. 하단 22px 세이프영역.

**House Card 구성** (`HouseCard`, 폭 326 × 높이 152):
- 좌: 사진 썸네일 110×128, 라운드 12. 좌상단 "방문일 방문" 어두운 뱃지, (사진은 placeholder — 실제 사용자 업로드 사진).
- 우: 상단 행 = 거래유형 뱃지(월세=coral soft / 전세=green soft) + 태그 뱃지(예: "햇빛 좋음") + 우측 하트(찜) 아이콘.
- 집 이름(15.5/700) → 주소·평수·구조·층(11/500, muted) → **가격**(19/700; 월세는 "1,000/65" + "관리비 7만", 전세는 "1.8억").
- 하단 행(구분선 위): 좌측 체크리스트 5개 아이콘(햇빛/수압/곰팡이/방음/환기, 각 좋음=green·보통=amber·나쁨=red 색) + 우측 **통근시간**(등록 시: 💼 그린 N분, 🎓 보라 M분; 미등록 시 표시 안 함).

**통근시간 표시 — 등록 상태별 (중요한 UX 결정):**
- 통근 기준(직장/학교)은 **지도 필터가 아니라 카드 정보**다. 그래서 검색바 근처엔 어떤 통근 UI도 두지 않는다.
- `commuteMode` = `'none' | 'work' | 'school' | 'both'`:
  - **none(미등록)**: 카드 위(리스트 영역)에 슬림 안내 배너 — "직장·학교 등록하고 **통근시간** 보기" + 우측 짧은 **"등록"** 버튼(ink). 카드 하단엔 통근시간 없음.
  - **work**: 카드에 💼 그린 N분 1개.
  - **school**: 카드에 🎓 보라 M분 1개.
  - **both**: 카드에 💼 그린 + 🎓 보라 2개 나란히.
- (모바일 카피 원칙: 길게 쓰지 않는다. 버튼은 "등록"처럼 짧게, 가능하면 아이콘만.)

**필터 시트** (`FilterSheet`): 필터 아이콘 탭 시 하단에서 올라오는 모달 시트(딤 배경 `rgba(14,26,20,0.35)`). 핸들 → "필터" 타이틀 + "초기화". 섹션:
- **거래 유형**: 세그먼트 토글(전체/월세/전세).
- **보증금**: 듀얼 핸들 레인지 슬라이더 + 우측 현재값("500 ~ 2,000만").
- **월세**: 듀얼 핸들 레인지 슬라이더("40 ~ 70만").
- **평수**: 칩(전체 / ~5평 / ~10평 / ~15평 / ~20평 / 20평~) — **5평 단위**.
- **방 개수**: 칩(원룸 / 1.5룸 / 투룸 / 쓰리룸+).
- 하단 액션: "닫기"(아웃라인) + "N개 매물 보기"(ink 채움).

**줌 레벨별 지도 표현** (`zoom` prop = `'far' | 'medium' | 'close'`):
- **far(줌아웃)**: 동 단위 **클러스터 핀**(물방울 핀 + 카운트 뱃지, size 46). 작은 라벨/보조도로 숨김.
- **medium(기본)**: 매물별 **가격 핀**(여기선 하우스 물방울 핀), 선택 시 콜아웃.
- **close(줌인)**: 건물 단위 **상세 핀**(이름+가격 박스), 보이는 매물 수 줄고 선택 시 콜아웃. 동네 보조 라벨(학교/역 등) 추가.
- 좌하단 줌 안내 뱃지(옵션): "지도 보기 · 동 단위/구역 단위/건물 단위".

### 2. 집 추가 (`pickhouse-add.jsx` → `AddWizard`)
**Purpose**: 매물 한 건의 정보를 입력. 필드가 많아 한 화면에 다 쏟지 않고 **4개 그룹(탭)** 으로 나눈다. 타이핑보다 토글·칩·스테퍼·트라이스테이트 위주.

**핵심 UX 결정:**
- **비선형**: 상단 탭(기본/가격/구조/체크)을 **눌러서 아무 순서로나 이동** 가능("순서 상관없이 원하는 항목부터 입력하세요"). 1단계부터 강제하지 않는다.
- 탭은 **언더라인 스타일**(텍스트 라벨 + 활성 항목 아래 슬라이딩 인디케이터, ink). 숫자/체크표시 없음.
- **기본 탭만 필수** → 탭 라벨 "기본" 옆에 coral 별표(`*`). 나머지는 선택.
- 하단 액션: 마지막(체크) 탭이면 "집 저장하기"(체크 아이콘), 그 외엔 좌측 "저장"(아웃라인, 언제든 저장 가능) + 우측 "다음 항목"(ink).
- 본문은 헤더와 하단바 사이 스크롤 영역(top≈174, bottom≈102).

**Step 1 — 기본:**
- 거래 유형: 세그먼트 토글(월세/전세). 선택 값에 따라 2단계 가격 입력 폼이 달라짐.
- 집 별칭: 텍스트 입력(집 아이콘).
- **주소**: "주소 검색" 버튼 → **전체화면 주소 검색 오버레이**(우편번호 API 스타일: 검색바 + 결과 리스트, 각 결과 = 우편번호 뱃지 + 도로명(ink 뱃지) + 지번(회색 뱃지) + 건물명). 결과 선택 시 → 잠긴 기본주소 카드(도로명+지번, "변경" 링크) + **상세주소 입력**(동/호수) 필드. 주소엔 보조 설명 문구 없음.
- 방문일: 텍스트(기본 오늘).

**Step 2 — 가격 (조건부 필드 핵심):**
- 거래유형이 월세면 **보증금 + 월세** 두 칸, 전세면 **전세금** 한 칸.
- 관리비: 금액 입력(없으면 0).
- **관리비 포함 항목**: 멀티 칩(수도 / 전기 / 가스 / 인터넷). *(청소 없음)*
- **조건부**: 포함에 **선택되지 않은** 전기·가스만 "별도 납부 (월 예상)" 금액 입력 폼이 나타난다. 칩으로 포함시키면 해당 입력은 사라진다.
- 하단 그린 요약 배너: "관리비 N만원에 **수도·인터넷** 포함 · **전기·가스** 별도 납부" (선택에 따라 실시간 갱신).

**Step 3 — 구조:**
- 방 구조: 단일 선택 칩(원룸 / 분리형 원룸 / 1.5룸 / 투룸 / 쓰리룸+).
- **평수**: 직접 입력 필드(전용면적, 큰 숫자 + "평" + **㎡ 자동 환산** 표시) + 자주 쓰는 평수 칩(6·9·12·15·18·24평). ※ +/- 스테퍼는 쓰지 않는다(20번 눌러야 해서).
- **층수**: **층 유형 세그먼트 토글(지상/반지하/옥탑)** 먼저 → 유형에 따라 아래 필드 변함. 지상=해당층/전체층 두 칸 숫자 입력, 반지하·옥탑=전체층 한 칸 + 확인 안내(채광·습기 / 단열·방수).
- 향: 칩(남향/남동향/남서향/동향/서향/북향). 남향 계열엔 햇빛 아이콘.

**Step 4 — 체크:**
- **컨디션 체크**: 햇빛·수압·곰팡이·방음·환기 각각 **트라이스테이트 토글(좋음=green / 보통=amber / 나쁨=red)**. 좌측 아이콘 색이 선택 값 따라 변함.
- 시설: 스위치 행(엘리베이터 / 주차 가능 / 풀옵션(빌트인)).
- 사진: 타일 그리드(추가 타일 "N/10" + 업로드된 사진 + 삭제 X). 최대 10장.
- 메모: 자유 텍스트(멀티라인).

### 3. 마이페이지 & 직장·학교 등록 (`pickhouse-places.jsx`)
**Purpose**: 통근 기준지(직장/학교)를 등록·관리. 등록 시 메인 카드와 (향후) 비교 화면에 통근시간이 자동 표시된다.

**진입점 결정**: 메인 검색바엔 통근 UI를 두지 않고, **마이페이지**가 등록 진입점이다(+ 메인 카드 위 미등록 안내 배너의 "등록" 버튼도 여기로).

**`MyPage`** (`registered` prop):
- 헤더 "마이" + 프로필 행(아바타 + 이름 + "기록한 집 7곳 · 찜 3곳").
- **통근 기준지 섹션**:
  - **미등록**(`registered=false`): 안내 카드 — route 아이콘 + "직장·학교를 등록해 보세요" + 설명("등록하면 집 카드와 비교 화면에 **통근시간**이 표시돼요...") + 그린 "직장·학교 등록" 버튼.
  - **등록됨**(`registered=true`): 직장 행(💼, "직장" 뱃지 + "주 통근지" ink 뱃지, **긴 이름** 예 "㈜카카오엔터프라이즈 판교 오피스" — 1줄 ellipsis) + 학교 행(🎓 보라, "학교" 뱃지, "한양대학교 서울캠퍼스 공과대학") + "장소 추가" 점선 버튼. 각 행 우측 edit 아이콘.
- 기타 메뉴 행: 찜한 집(3) / 내가 기록한 집(7) / 비교 목록(4) / 설정.
- 하단 탭바(마이 활성).

**`PlacesList`** (등록된 장소 목록 전용 화면): why 배너(그린) + "등록된 장소 N" + `PlaceCard` 리스트 + "장소 추가" + 하단 "완료".

**`PlaceCard`**: 좌 아이콘 박스(직장=green / 학교=보라 / 기타=coral) + 유형 뱃지 + (주 통근지 뱃지) + **이름(긴 이름 2줄 line-clamp)** + 주소(1줄 ellipsis) + edit. 하단(구분선): 이동수단 칩("대중교통 기준" 등) + "매물별 통근시간 자동 계산".

**`AddPlace`** (새 장소 추가 — 카카오 검색 핵심):
- 장소 종류: 칩 카드 3개(직장/학교/기타, 각 색상).
- **회사·학교 검색**: 주소가 아니라 **장소(회사/학교) 이름**으로 검색. "kakao 검색" 버튼(노란 카카오 뱃지) → **전체화면 카카오 장소 검색 오버레이**(`KakaoPlaceOverlay`): 결과 = 장소명(긴 이름 ellipsis) + 카테고리 뱃지 + 도로명 주소 + 거리. 선택 시 → 선택 장소 카드(카테고리 + 긴 이름 2줄 clamp + 주소, "변경").
- **표시 이름(별칭)**: 자동 채움 + 편집 가능. "카드엔 아이콘만 표시 · 짧게 권장" 안내. (긴 회사명이어도 메인 카드는 아이콘+시간만 쓰므로 레이아웃 안 깨짐.)
- 주 이동수단: 칩 카드 4개(대중교통/자동차/도보/자전거). 통근시간 계산 기준.
- 주 통근지 토글 스위치(켜면 메인 카드에 이 장소 시간 표시).
- 하단 "장소 저장하기".

**`PlacesEmpty`**: 온보딩 빈 상태(중앙 route 일러스트 + "통근 기준지를 등록해 보세요" + 설명 + 그린 "직장·학교 등록" 버튼 + "나중에 할게요").

## Interactions & Behavior
- **카드 ↔ 핀 동기화**: 카드 캐러셀 가로 스크롤 → snap 인덱스 계산 → 해당 매물 핀이 콜아웃으로 전환(반대로 핀 탭 시 카드도 이동). 구현 시 스크롤/제스처와 지도 selection을 단일 상태로 묶을 것.
- **줌**: 지도 SDK 줌 레벨 임계값에 따라 far/medium/close 핀 렌더 전환 + 클러스터링.
- **필터/시트/오버레이**: 하단 모달 시트(드래그 핸들), 전체화면 검색 오버레이(주소·카카오). 딤 탭으로 닫기.
- **집 추가 비선형 탭**: 탭 클릭으로 어느 스텝이든 즉시 이동, 저장은 어느 시점에나 가능.
- **조건부 필드**: (가격) 거래유형→가격 입력 형태, 관리비 포함칩→별도납부 입력 표시; (구조) 층 유형→하위 필드 전환.
- **트라이스테이트/스위치/스테퍼/세그먼트** 등 컨트롤 상태는 `pickhouse-add.jsx`의 각 컴포넌트(`Segmented`, `Chips`, `PyeongInput`, `FloorInput`, `TriState`, `SwitchRow`, `DirectionPicker`) 구현을 그대로 참조.
- 접근성/터치: 모든 탭 타깃 최소 44px. 텍스트 12px 이상(뱃지 제외).

## State Management
- 메인: `selectedHouseId`(= 활성 카드 인덱스), `zoomLevel`, `filter`(거래유형/보증금범위/월세범위/평수/방수), `commuteMode`('none'|'work'|'school'|'both' — 등록된 기준지에서 파생).
- 집 추가: `step`(1~4), `txType`, 가격(보증금/월세/전세금/관리비/포함항목 배열/별도납부), 구조(rooms/pyeong/floorType/floor/dir), 체크(컨디션 객체/시설 객체/사진/메모), 주소(기본주소 객체/상세).
- 장소: 등록 장소 배열(kind/name/addr/transport/primary), 카카오 검색 오버레이 open 상태.
- 데이터 페칭(실서비스): 주소 검색 API(도로명/지번), **카카오맵 장소 검색 API**(키워드), 통근시간 계산(경로 API — 이동수단별 소요시간).

## Data Model (참고 — `pickhouse-shared.jsx`의 HOUSES)
각 매물: `{ id, name, addr, type('rent'|'jeonse'), deposit, rent, mgmt, pyeong, rooms, floor, visited, x/y(지도좌표→실서비스는 lat/lng), cl:{sun,water,mold,noise,vent (1~3)}, tag, note, commute(직장 분), commuteSchool(학교 분) }`. 점수/별점 필드는 **의도적으로 없음**(사람마다 집 기준이 달라 점수화하지 않기로 결정).

## Assets
- 이미지/아이콘 외부 에셋 없음. 아이콘은 인라인 SVG(대체 가능). 지도는 가짜 SVG → 실 SDK로 교체. 사진은 사용자 업로드(현재 placeholder 그라데이션).
- 폰트: Pretendard(웹 CDN 또는 네이티브 임베드).

## 구현 권장 순서
1. 디자인 토큰 + 타이포 셋업 → 2. 공통 컴포넌트(뱃지/칩/세그먼트/시트/입력 컨트롤) → 3. 메인(지도 SDK + 핀/콜아웃 + 카드 캐러셀 + 동기화 + 탭바) → 4. 필터 시트 → 5. 집 추가 위저드(조건부 필드) → 6. 마이/직장·학교 등록(카카오 검색).
